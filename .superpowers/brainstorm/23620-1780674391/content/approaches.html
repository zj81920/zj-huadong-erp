三种实现方案对比

均基于：传统分页 + 每页条数可选(20/50/100) + 筛选联动 + 全量改造

A

共享 Hook + 组件（推荐）

创建 usePagination hook + PaginationBar 组件。每个页面引入 hook 管理状态，引入组件渲染分页栏。

优势

  • 改动最小，每个页面只需替换部分代码
  • 灵活性高，各页面仍可自己控制表格列
  • 不改变现有页面结构

缺点

  • 每个页面仍需写 fetch 调用代码
  • hook 用法需保持一致
B

容器组件模式

创建 <PaginatedList> 容器，传入 fetch 函数 + 列定义，容器统一管理分页、加载、空状态。

优势

  • 极简页面代码，只需传配置
  • 分页、加载态、空状态统一处理

缺点

  • 侵入性强，现有页面需大规模重构
  • 复杂列表（多 Tab、特殊列）适配困难
C

URL 参数驱动

分页参数存入 URL query string（?page=1&pageSize=20），利用 Next.js useSearchParams 驱动。

优势

  • 刷新保留分页位置
  • 可分享带分页状态的链接

缺点

  • 引入额外复杂度
  • ERP 场景极少需要分享分页链接
