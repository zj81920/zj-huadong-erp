列表行底色设计方案

柔和整行底色 + 集中颜色映射 + 通用 row class

颜色映射（全局集中）

状态行底色CSS Class
草稿#FAFAF9 默认白.row-status-draft
审批中#FEF9E7 极淡黄.row-status-pending
已批准#F0FDF4 极淡绿.row-status-approved
已驳回#FEF2F2 极淡红.row-status-rejected
已归档/已生效#F0F4FF 极淡蓝.row-status-archived
逾期#FFF7ED 极淡橙.row-status-overdue

各业务特有状态（如「当前有效」「执行」「未收」等）按语义归类到上述 6 种中。

实现方式

只需 2 个文件改动就可以覆盖全部页面：

src/lib/status-colors.ts — 集中映射 
// 状态 → row class 映射（全局唯一）
export const STATUS_ROW_COLORS: Record<string, string> = {
  "草稿":     "row-status-draft",
  "审批中":   "row-status-pending",
  "已批准":   "row-status-approved",
  "已驳回":   "row-status-rejected",
  "已归档":   "row-status-archived",
  "合同归档": "row-status-archived",
  "已生效":   "row-status-archived",
  // ... 其他状态归类
};

// 工具函数：根据状态获取行 class
export function getRowStatusClass(status: string | null): string {
  return status ? (STATUS_ROW_COLORS[status] ?? "") : "";
}
src/app/globals.css — 新增 6 个 row class 
/* 行底色区分 */
.row-status-pending  { background: #FEF9E7; } /* 审批中 - 极淡黄 */
.row-status-approved { background: #F0FDF4; } /* 已批准 - 极淡绿 */
.row-status-rejected { background: #FEF2F2; } /* 已驳回 - 极淡红 */
.row-status-archived { background: #F0F4FF; } /* 已归档 - 极淡蓝 */
.row-status-overdue  { background: #FFF7ED; } /* 逾期 - 极淡橙 */
.row-status-draft    { background: #FAFAF9; } /* 草稿 - 默认白 */

页面使用（一行代码）

// 在每个列表页面中，给 <tr> 加一个 class
{items.map(item => (
  <tr key={item.id} className={getRowStatusClass(item.status)}>
    ...
  </tr>
))}

特殊场景

场景处理方式
无 status 模块不传 className，白色默认
待办-待处理按优先级：紧急→淡红、重要→淡黄、普通→白
待办-已处理按审批状态正常映射
hover 效果保持原有 hover 浅灰，叠加在底色上
