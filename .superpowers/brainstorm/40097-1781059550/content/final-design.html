完整设计方案 — 项目WBS计划与进度

页面结构

入口流程
项目列表
选择项目 → 点击"计划"
→
WBS计划页
该项目的WBS树 + 进度

关键变更汇总

维度 设计
页面定位 项目列表页每行增加"计划"按钮 → 进入该项目专属的WBS计划页面
路径: /projects/plans/{projectSourceId}
WBS层级 一级(设计阶段) → 二级(子项) → 三级(专业) → 四级(任务)
一级节点来源 项目立项时勾选的设计阶段（方案设计/初步设计/详细设计/竣工图设计 + 采购可选）
二级节点 用户在WBS计划页手动创建子项
三级节点 从专业字典选择（系统设置中维护）
四级节点 手动创建任务，填报计划%/实际%/负责人/状态
进度汇总 4级任务填报 → 自动汇总到3级专业 → 2级子项 → 1级设计阶段
里程碑 作为节点的标记属性，可在任意层级节点上打标记
专业字典 新建 discipline_dictionary 表，系统设置中维护
原菜单 侧边栏"项目计划"和"项目进度"合并为"项目WBS计划"，去掉"项目进度"菜单
删除冗余 删除原 project_progress 表和原扁平 project_plans 表

数据库 — 新建表

// 1. 专业字典表
model DisciplineDictionary {
id String @id @default(cuid())
code String @unique // 专业编码: PROC, PIPE, ELEC...
name String // 专业名称: 工艺, 管道, 电气...
sortOrder Int @default(0)
isActive Boolean @default(true)
}

// 2. WBS节点表（树形结构，替代原 ProjectPlan）
model ProjectWbsNode {
id String @id @default(cuid())
projectSourceId String // 关联项目
parentId String? // 父节点ID，null=一级节点
level Int // 1=设计阶段, 2=子项, 3=专业, 4=任务
name String // 节点名称
disciplineCode String? // 三级节点关联的专业编码
isMilestone Boolean @default(false) // 里程碑标记
plannedPct Int @default(0) // 计划进度%
actualPct Int @default(0) // 实际进度%（汇总或手动填）
delayDays Int @default(0) // 滞后天数
alertStatus String @default("正常")
responsibleId String? // 负责人
status String @default("未开始") // 未开始/进行中/已完成
startDate DateTime?
endDate DateTime?
sortOrder Int @default(0)
version Int @default(1)
}
