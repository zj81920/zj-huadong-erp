主要变化总结

改了什么

  • "进度"列 → "计划 vs 实际"双进度条（上下对比）
  • 新增"偏差"列：显示滞后天数 + 计划/实际差值百分比
  • 统计卡片增加"滞后"预警卡片（红色呼吸灯）
  • 筛选条件增加"预警状态"下拉框（全部/正常/滞后）
  • 滞后行高亮背景（浅红色背景）
  • 数据库：ProjectPlan 增加 plannedPercentage / delayDays / alertStatus 字段

没改什么

  • 表单页（新建/编辑）基本不变，只加"计划进度%"字段
  • 详情弹窗增强（增加双进度展示）
  • API 路由改造小——新增字段写入+alertStatus自动计算
  • 删除独立的 project_progress 表和相关代码
  • 侧边栏去掉"项目进度"菜单项

这就是方案A的具体效果。 改动集中在现有页面的几个列上，不改变页面整体布局和交互模式。 你觉得这个设计符合预期吗？或者需要调整什么地方？
