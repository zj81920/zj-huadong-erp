外包任务审批通过后回写 WBS 责任人，删除后恢复原状

数据模型补充

1. WBS responsibleIds 格式变更（支持外包标识） 
// 当前格式（纯 ID 数组）
responsibleIds: ["user_id_1", "user_id_2"]

// 新格式（结构化对象数组，向后兼容）
responsibleIds: [
  { type: "person", id: "user_id_1", name: "张三" },
  { type: "person", id: "user_id_2", name: "李四" },
  { type: "outsourcing", id: "outsourcing_task_id", name: "华东设计院" }
]
2. 审批通过时回写逻辑
  1. 审批状态变为"已批准"时触发
  2. 遍历该外包的 OutsourcingWbsItem 明细
  3. 在每个 WBS 节点的 responsibleIds 中追加 {type:"outsourcing", id:..., name:外包对象名}
  4. 如果已被驳回后重新批准，先清理旧的 outsourcing 条目再写入
  5. 审批驳回 → 不写入 WBS，WBS 保持原状但继续锁定
3. 删除外包时恢复逻辑
  1. 删除外包任务时触发
  2. 遍历关联的 WBS 节点，从 responsibleIds 中移除所有 type=outsourcing 且 id 匹配的条目
  3. 删除 OutsourcingWbsItem 关联记录
  4. WBS 节点恢复为"无责任人"状态，可被其他外包重新选择
4. 前端 WBS 责任人展示
WBS 树中责任人列展示时，type=outsourcing 条目显示外包对象名称 + 外包 标签， 区别于普通人员。
