三种实现方案对比

请选择您倾向的方案

A

改良 checkbox 列表

保留现有 checkbox 列表形式，但默认全部勾选在职员工

优点

  • 改动最小，前端改动少
  • 员工可见，可逐个取消

缺点

  • 员工多时滚动列表仍显笨重
  • 没有搜索/筛选功能
B

摘要卡片 + 排除名单

主体显示汇总信息（人数、总额），排除操作在独立弹窗中完成

优点

  • 新建弹窗简洁清爽
  • 排除操作有独立空间
  • 可加入搜索/按部门筛选

缺点

  • 需额外弹窗操作
  • 开发工作量中等
C

内联表格（推荐）

新建弹窗直接展示员工明细表格，所有在职员工默认已有数据，可编辑金额也可删除整行排除

优点

  • 一步完成：选择+编辑金额
  • 直观看到每个人薪酬
  • 删除行 = 排除员工

缺点

  • 员工多时表格可能很长
  • 开发工作量稍大

我的推荐：方案 C（内联表格）

理由：当前新建批次后紧接着就是编辑明细的步骤。如果把两步合并，用户在新建时就能看到所有员工的薪酬预览并直接编辑，同时可以通过"删除行"来排除员工，流程更顺畅。
