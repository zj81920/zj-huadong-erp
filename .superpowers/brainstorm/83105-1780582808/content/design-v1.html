统一设计方案 v1

方案 A 的详细设计，按"统一查看标准"和"统一操作栏标准"展开

标准一：统一的"查看"按钮和弹窗

按钮标准
👁 查看
  • 所有模块统一使用 "查看" 文案
  • 统一使用 Eye 图标
  • 始终显示在操作栏第一位
弹窗内容标准
✅ 包含
  • 完整的业务信息（只读）
  • 关联单据列表（发票等）
  • 审批区块（如有审批实例）
❌ 不包含
  • 编辑/删除按钮
  • 登记发票/开票按钮
  • 提交审批/发起变更
  • 任何数据写入操作

标准二：统一的"操作栏"布局

操作栏按钮排序标准（从左到右）
1. 👁 查看 2. ✏️ 编辑 3. 🗑 删除 4. 📄 登记发票/开票 5. 🔄 发起变更 6. ➡️ 提交审批 7. ➡️ 前往审批中心 8. 📁 合同归档
说明：
  • 1-3 为基础操作，所有模块都应有
  • 4-5 为业务关联操作，按模块实际需求显示
  • 6-8 为审批流相关操作，仅在启用了审批流且有权限时显示
  • 按钮通过权限控制(rolePerms)和状态控制(approvalStatus)决定是否显示

标准三：审批流状态的操作标准

当前状态 操作栏按钮 查看弹窗内容 说明
草稿 查看 + 编辑 + 删除 + 提交审批 完整业务信息（无审批区块） 可编辑可删除
审批中 查看 + [编辑/删除有条件] + 前往审批中心 完整业务信息 + 审批区块 编辑/删除由权限控制
已批准 查看 + [编辑/删除有条件] + 合同归档（如有） 完整业务信息 + 审批区块（已通过） 归档后才可发起变更
已驳回 查看 + 编辑 + 删除 + 提交审批 完整业务信息 + 审批区块（含驳回原因） 退回草稿状态，可重新提交

标准四：审批中心视图标准

审批中心打开"处理审批"时，直接复用"查看"的完整内容 + 审批操作按钮。
  • 上半部分 = 完整的业务信息（和"查看"弹窗一模一样）
  • 下半部分 = 审批时间线（当前节点高亮）
  • 底部 = 通过/驳回按钮（当前审批人可见）
