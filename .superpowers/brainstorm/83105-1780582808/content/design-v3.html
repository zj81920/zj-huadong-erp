最终设计方案 v3

所有模块统一由流程配置驱动，无节点=纯CRUD，有节点=审批流

核心设计理念

流程设置 → 配置了 N ≥ 1 个节点？ → ✅ 有审批流 或者 → ❌ 纯CRUD

规则矩阵

模块状态 操作栏 "查看"弹窗内容
纯CRUD（无节点） 查看 + 编辑 + 删除 只含业务信息
审批流（有节点）
— 草稿		 查看 + 编辑 + 删除 + 提交审批 只含业务信息（待提交后才产生审批实例）
审批流（有节点）
— 审批中		 查看 + [编辑/删除有条件] 业务信息 + 审批区块（时间线只读）
审批流（有节点）
— 已批准		 查看 + [编辑/删除有条件] 业务信息 + 审批区块（已通过）
审批流（有节点）
— 已驳回		 查看 + 编辑 + 删除 + 提交审批 业务信息 + 审批区块（含驳回原因）

审批中心视图

审批中心打开审批时，直接展示和"查看"弹窗 完全一样 的业务内容，底部加上审批操作按钮（通过/驳回）。
