设计 Section 1：简化数据模型

去冗余、保核心，让模型自解释

❌ 当前模型（复杂）
planStartDate     DateTime?   ← 保留
planEndDate       DateTime?   ← 保留
actualStartDate   DateTime?   ✕ 删除
actualEndDate     DateTime?   ✕ 删除
progress          Int         ← 保留
status            String      ✕ 删除（不再存储）
delayDays         Int         ✕ 删除（不再存储）
isMilestone       Boolean     ← 保留
responsibleId     String?     ← 保留
version           Int         ← 保留
✅ 新模型（简约）
planStartDate     DateTime?   ← 计划开始
planEndDate       DateTime?   ← 计划结束
progress          Int         ← 0-100 手动填
isMilestone       Boolean     ← 里程碑标记
responsibleId     String?     ← 责任人
version           Int         ← 乐观锁

删除 3 个字段：
actualStartDate / actualEndDate /
status / delayDays

+ aiGenerated      Boolean?   ← 新增
  (标记 AI 生成的任务)

关键变化说明

🚫 去掉 actualStart/End

实际进度只看 progress% 和计划日期。开始 = progress > 0，完成 = progress = 100。不再需要单独维护实际日期。

🚫 去掉 status / delayDays

状态和延误由前端纯函数实时计算，不存数据库。消除前后端两套判断逻辑的问题。

✅ 新增 aiGenerated

标记由 AI 自动生成的任务节点，方便区分人工创建和 AI 创建的节点。后续 AI 更新时可识别。

✅ 状态实时计算

基于 progress + planStart/End + today，纯函数计算：未开始 / 进行中 / 已完成，以及是否延误。
