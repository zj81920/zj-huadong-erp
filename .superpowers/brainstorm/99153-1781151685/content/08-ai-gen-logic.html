AI 自动生成 L2-L4 的逻辑

基于项目上下文 + 专业字典的智能展开

生成流程

输入 项目类型(设计/EPC/咨询) + 阶段名称(如"初步设计") + 专业字典(DisciplineDictionary)
处理 调用 LLM，携带系统提示词（行业知识），返回结构化 JSON
输出 完整的 L2→L3→L4 树，包含名称 + 建议工期 + 责任人建议

L2 和 L3 的关系是什么？

我的理解是 L2 = 专业大类(建筑/结构/给排水...)，L3 = 该专业下的细分。但需要你确认实际的层级划分惯例。

A

L2=专业大类 L3=子专业

如 L2「建筑专业」→ L3「建筑设计」「建筑详图」；L3 关联 DisciplineDictionary

B

L2=交付物/区域 L3=专业

如 L2「A区图纸」→ L3「建筑」「结构」「给排水」；L3 关联 DisciplineDictionary
