补充：提前开始的判断

延误 / 正常 / 提前 — 三档一目了然

完整状态判断逻辑

planPct = clamp((today - planStart) / (planEnd - planStart) × 100, 0, 100)
对比 progress 和 planPct：
🚀
提前

已开始但还未到计划开始日
或 progress > planPct
进度领先于计划

✅
正常

progress ≈ planPct
或 progress = 0 且未到开始日
一切按计划进行

⚠️
延误

progress < planPct
或已过计划结束日但未完成
需要关注

🏁
已完成

progress = 100
无论是否超期
任务结束

具体场景举例

场景条件状态
提前开工计划8月开始，今天6月，progress=20%🚀 提前
进度领先计划1~10月，今天6月(planPct=50%)，progress=70%🚀 提前
按计划推进计划1~10月，今天6月(planPct=50%)，progress=50%✅ 正常
等待开始计划8月开始，今天6月，progress=0✅ 正常
进度滞后计划1~10月，今天6月(planPct=50%)，progress=30%⚠️ 延误
超期未完成计划1~5月，今天7月，progress=80%⚠️ 延误
已完成progress=100（无论是否超期）🏁 已完成

一个纯函数，四种状态

// 输入: progress, planStart, planEnd, today
// 输出: { status, label }
if (progress === 100) → 🏁 已完成
if (progress === 0 && today < planStart) → ✅ 正常
if (progress > 0 && today < planStart) → 🚀 提前
if (progress >= planPct) → 🚀 提前
else → ⚠️ 延误
