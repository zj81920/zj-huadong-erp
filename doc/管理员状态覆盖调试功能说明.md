# 管理员状态覆盖调试功能说明

> **用途**：开发调试期间，管理员可手动切换任意业务单据的审批状态，快速验证完整流程。
> **重要**：部署生产环境前，需移除此功能（见底部清理清单）。

---

## 二、支持的业务类型与副作用

管理员选择状态后，API 会自动触发与业务 API 一致的副作用逻辑：

| 业务类型 | 管理员可选状态 | 自动触发的副作用 |
|---------|--------------|----------------|
| purchase_request | 草稿/审批中/已批准/已驳回/已转询价/已采购 | 无 |
| inquiry | 草稿/审批中/已批准/已驳回 | 无 |
| quotation | 草稿/审批中/已批准/已驳回 | 无 |
| **income_contract** | 草稿/审批中/已批准/已驳回/合同归档 | 批准时自动创建应收记录（支持分期） |
| **expense_contract** | 草稿/审批中/已批准/已驳回/合同归档 | 批准时自动创建应付记录 + 关联采购需求更新为"已采购" |
| non_contract_income | 草稿/审批中/已批准/已驳回 | 无 |
| non_contract_expense | 草稿/审批中/已批准/已驳回 | 无 |
| **outsourcing** | 草稿/审批中/已批准/已驳回 | 分包公司批准时自动生成支出合同草稿 + 回填 contractId |
| payment_application | 草稿/审批中/已批准/已驳回/已付款 | 无 |
| expense_report | 草稿/审批中/已批准/已驳回 | 无 |
| **other_borrowing** | 草稿/审批中/已批准/已驳回/未还清/已还清 | 批准时自动映射为"未还清" |
| **lending_out** | 草稿/审批中/已批准/已驳回/未还清/已还清 | 批准时自动映射为"未还清" |
| salary_payment | 草稿/审批中/已批准/已驳回/已发放 | 无 |

### 防重复保护

所有副作用均具备双重保护：

1. **状态变化判断**：只有从非"已批准"变为"已批准"才触发，已是"已批准"再选不重复触发
2. **已有记录检查**：即使状态判断通过，也会检查是否已存在关联数据（应收/应付/合同），防止数据重复
