## 一、交易对方搜索（问题 1）

- [ ] **1.1** 创建/编辑其他支出弹窗中，交易对方字段是搜索组件（CounterpartySearch），不是普通 input
- [ ] **1.2** 输入文字时出现下拉建议列表，数据来源为往来信息库（`/api/counterparty?search=...`）
- [ ] **1.3** 选中建议项后，开户行和银行账号自动回填到对应字段
- [ ] **1.4** 也可以手动输入新名称（不在往来信息库中的），不强制选择建议项

## 二、往来信息保存（问题 2）

- [ ] **2.1** 非管理员用户创建其他支出时，交易对方信息成功保存到往来信息库
- [ ] **2.2** 重复的往来信息不会重复创建（name + bankName + bankAccount 去重）
- [ ] **2.3** 只有名称没有银行信息时也能正常保存
- [ ] **2.4** 审批通过后，往来信息也会同步到往来信息库（approval-engine.ts 终态逻辑）

## 三、详情页银行信息（问题 3）

- [ ] **3.1** 其他支出详情弹窗中显示"对方银行信息"区域（包含开户行和账号）
- [ ] **3.2** 当没有银行信息时，不显示该区域
- [ ] **3.3** 开户行和账号都能正确显示

## 四、发票登记与状态管理（问题 4）

### 4.1 创建支出时录入发票

- [ ] **4.1.1** 创建其他支出弹窗底部有"发票信息"区域，包含"有发票"开关
- [ ] **4.1.2** 开关关闭时（默认），不显示发票录入表单
- [ ] **4.1.3** 开关打开时，显示发票号码、开票日期、金额、税率输入字段
- [ ] **4.1.4** 填写发票信息后提交，支出记录和发票记录同时创建成功
- [ ] **4.1.5** 创建时发票的 `sourceType` = `non_contract_expense`，`sourceId` 正确关联

### 4.2 待补票标记

- [ ] **4.2.1** 创建支出时不勾选"有发票"且不标记"无需开票"，`invoiceStatus` 应为"待补票"
- [ ] **4.2.2** 其他支出列表中，发票状态列显示"待补票"徽章（黄色）
- [ ] **4.2.3** 详情弹窗中显示发票状态和"补录发票"按钮

### 4.3 补录发票

- [ ] **4.3.1** 点击"补录发票"按钮，弹出补录发票弹窗
- [ ] **4.3.2** 填写发票号码和日期后提交，发票创建成功
- [ ] **4.3.3** 补录成功后，支出的 `invoiceStatus` 自动变为"已收票"
- [ ] **4.3.4** 补录成功后，`invoicedAmount` 自动更新

### 4.4 发票管理页

- [ ] **4.4.1** 发票管理页筛选栏新增"待补票"筛选项
- [ ] **4.4.2** 待补票状态的发票显示黄色徽章
- [ ] **4.4.3** 每条"已登记"发票操作列新增"作废"按钮
- [ ] **4.4.4** 点击"作废"后确认，发票状态变为"已作废"（红色徽章）
- [ ] **4.4.5** 作废发票后，关联模块的 `invoicedAmount` 自动扣减
- [ ] **4.4.6** 已作废发票不能再次编辑（API 返回 400）
- [ ] **4.4.7** 已作废发票删除时不会重复扣减 `invoicedAmount`
