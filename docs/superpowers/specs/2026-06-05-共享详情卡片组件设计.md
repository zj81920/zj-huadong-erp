# 共享 DetailCard 组件体系设计

## 背景

当前应用中，审批页面和模块详情页有两套独立的详情展示代码：

- **审批页面** (`src/app/(dashboard)/approvals/page.tsx`)：内联定义了 18 个 `*DetailCard` 组件，共约 650 行
- **各模块详情弹窗**（收入合同、支出合同、供应商、发票等 6+ 处）：各自手写字段渲染代码

两套代码展示相同的数据字段，但独立维护，容易产生差异。

## 目标

将详情展示代码提取为共享组件，实现：

1. **一处修改，两处同步** — 模块页面和审批页面引用同一套 DetailCard 组件
2. **新增字段只需改一个文件** — 修改对应的 `XxxDetailCard.tsx`，两处自动生效
3. **新增流程模块时有标准模式可循** — 创建 `XxxDetailCard.tsx` + 在注册中心注册

## 设计决策

- **字段展示完全一致** — 审批页面和模块页面展示相同的字段集，布局也保持一致
- **DetailCard 纯展示** — 不包含任何操作按钮（编辑、删除、审批操作），由调用方负责包装
- **不承担数据获取** — 只接受 `data` prop，由调用方负责查询

## 组件设计

### DetailGrid（基础布局组件）

```tsx
interface DetailGridProps {
  fields: { label: string; value: React.ReactNode }[]
  columns?: 1 | 2  // 默认 2 列
}
```

从 `approvals/page.tsx` 的 `DetailGrid`（L780-791）直接提取，代码不变。

### DetailCard 组件模板

每个 DetailCard 遵循以下模式：

```tsx
interface Props {
  data: PrismaModelType  // 使用 Prisma 生成的类型，不用 any
}

export function XxxDetailCard({ data }: Props) {
  return (
    <div className="space-y-6">
      <DetailGrid fields={[...]} />
      {/* 有子表时在此处渲染 */}
    </div>
  )
}
```

### 注册中心（index.ts）

```tsx
export const DETAIL_CARD_MAP: Record<string, React.ComponentType<{ data: any }>> = {
  supplier: SupplierDetailCard,
  income_contract: IncomeContractDetailCard,
  // ...
}
```

## 使用方式

### 模块页面

```tsx
import { SupplierDetailCard } from '@/components/detail-cards'

// 在 Modal 中使用
<Modal isOpen={showDetail} onClose={() => setShowDetail(false)} title="供应商详情">
  <SupplierDetailCard data={detailSupplier} />
  <div className="flex justify-end gap-3 mt-6">
    <Button onClick={handleEdit}>编辑</Button>
  </div>
</Modal>
```

### 审批页面

```tsx
import { DETAIL_CARD_MAP } from '@/components/detail-cards'

const CardComponent = DETAIL_CARD_MAP[businessType]
// 在 BusinessDetailPanel 中使用
<CardComponent data={approvalData} />
```

## 新增流程模块的标准步骤

1. 在 `src/components/detail-cards/` 下创建 `XxxDetailCard.tsx`
2. 在 `detail-cards/index.ts` 的 `DETAIL_CARD_MAP` 中添加映射
3. 模块页面的详情弹窗引用该组件
4. 审批页面自动通过 `DETAIL_CARD_MAP` 获取该组件

## 实施顺序

1. 创建 `DetailGrid.tsx` + `index.ts`（基础设施）
2. 用 `SupplierDetailCard` 做试点，验证两处引用
3. 批量提取剩余 17 个 DetailCard 组件
4. 各模块页面详情弹窗改为引用共享组件
5. 审批页面删除内联的 DetailCard 定义，改为引用 `DETAIL_CARD_MAP`
6. 更新项目规则文件
