# 批量应用审批流程修复设计

## 背景

流程设置页面（`settings/approval-flow/page.tsx`）中，"批量应用到其他模块"功能存在但无法正常工作。

### 当前状态

- **UI 完整**：按钮、弹窗、模块多选列表、确认按钮均已实现
- **功能无效**：点击"确认应用"后，请求发送到后端返回 400 错误，而前端错误地显示"批量应用成功"

### 根因

前端发送的数据格式与后端 API 期望的格式**完全不匹配**：

| 层面 | 前端实际发送 | 后端 API 期望 |
|------|-------------|--------------|
| 顶层 key | `{ items: [...] }` | `{ sourceBusinessType, sourceFlowLevel, targets: [...] }` |
| 每个目标 | 含完整 `nodes` 数组 | 只有 `businessType` + `flowLevel` |
| 源数据来源 | 内存中的 `nodes` 状态 | 从数据库读取 |

此外，前端未检查 `res.ok`，导致请求失败时仍显示"成功"。

## 修复方案（方案一：修复前端匹配后端）

### 修复内容

**1. 前端 `handleBatchConfirm` 函数**

将 `page.tsx` L284-309 中的 `handleBatchConfirm` 改为：

```tsx
const handleBatchConfirm = async () => {
  setBatchApplying(true);
  try {
    const targets: { businessType: string; flowLevel: string }[] = [];
    Object.entries(batchTargets).forEach(([modType, levels]) => {
      levels.forEach((level) => {
        targets.push({ businessType: modType, flowLevel: level });
      });
    });

    const res = await fetch("/api/approval-flows/apply", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        sourceBusinessType: selectedModule,
        sourceFlowLevel: "common",
        targets,
      }),
    });

    if (!res.ok) {
      const err = await res.json();
      throw new Error(err.error || "批量应用失败");
    }

    setBatchModalOpen(false);
    setSaveMsg({ type: "success", text: "批量应用成功" });
  } catch (e) {
    const msg = e instanceof Error ? e.message : "批量应用失败，请重试";
    setSaveMsg({ type: "error", text: msg });
  } finally {
    setBatchApplying(false);
  }
};
```

关键改动：
- 发送格式改为 `{ sourceBusinessType, sourceFlowLevel, targets }`
- 添加 `res.ok` 检查，失败时抛出错误显示后端返回的错误消息
- 错误时保持弹窗打开，让用户看到错误信息后自行关闭

**2. 单元测试**

新增 `test/api/approval-flows-apply.test.ts`，覆盖：
- 正常复制流程：源模块有流程节点，成功复制到目标模块
- 源模块不存在：返回 404
- 目标模块有审批中实例：返回 409

## 无修改的文件

- 后端 `apply/route.ts` — 保持原样，已有完整逻辑
- Prisma Schema — 无需变更
- 其他 API 路由 — 无需变更

## 注意事项

- 用户必须**先保存**当前模块的流程，再批量应用（后端从 DB 读取源流程节点）
- 如果源模块没有配置流程节点，后端返回 404
- 如果目标模块有审批中的实例，后端返回 409，弹窗不会关闭
